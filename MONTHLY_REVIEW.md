## Общий результат

За первый месяц обучения я прошёл путь от базовых конструкций Python до создания полноценного учебного CLI-приложения для управления состояниями сервисов.

Проект постепенно развивался: сначала данные хранились непосредственно в коде, затем было добавлено сохранение в JSON, обработка повреждённых файлов, автоматические тесты, логирование, argparse, GitHub Actions, виртуальное окружение и Makefile.

## Возможности CLI-проекта

Разработанное CLI-приложение позволяет управлять состояниями учебных сервисов через командную строку.

Поддерживаются команды:

- `list` — показать все сервисы и их текущие состояния;
- `status <service>` — показать состояние выбранного сервиса;
- `start <service>` — перевести сервис в состояние `running`;
- `stop <service>` — перевести сервис в состояние `stopped`.

Приложение также умеет:

- хранить состояния сервисов в JSON-файле;
- загружать данные при запуске;
- сохранять изменения после выполнения команд;
- восстанавливать стандартные данные при отсутствии файла;
- обрабатывать повреждённый JSON;
- проверять существование сервиса;
- выводить понятные сообщения об ошибках;
- записывать действия и ошибки в лог-файл;
- возвращать корректные коды завершения;
- запускаться через команды `make`.

- работы при отсутствии файла;
- обработки повреждённого JSON;
- проверки неверного типа данных;
- отображения списка сервисов;
- получения статуса сервиса;
- запуска и остановки сервисов;
- обработки неизвестного сервиса;
- проверки аргументов командной строки.

Для уменьшения повторяющегося кода применяются `pytest fixtures`.

Актуальные тесты находятся в каталоге `day26/tests`. Их можно запустить командой:

make test
